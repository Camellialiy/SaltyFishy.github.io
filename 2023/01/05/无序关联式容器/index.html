无序关联式容器

概念

无序关联式容器，又称哈希容器，是一种不会自动排序的、可以快速查找。删除、修改的容器。

和关联式容器一样，无序容器也使用键值对（pair 类型）的方式存储数据

关联式容器和无序关联式容器的底层实现存在区别：

  • 关联式容器的底层实现采用的是红黑树
  • 无序容器的底层实现采用的是哈希表

无序关联式容器和关联式容器在使用上的区别在于：

1.无序容器内部存储的键值对是无序的，各键值对的存储位置取决于该键值对中的键

2.和关联式容器相比，无序容器擅长通过指定键查找对应的值（时间复杂度可近似认为 O(1)）；但对于使用迭代器遍历容器中存储的元素，无序容器的执行效率则不如关联式容器

以上两个区别，都是因为无序关联式容器的底层实现（哈希表）采取的是链地址法

如果是map，将数字改为键值对即可。

STL标准库在创建哈希表时，首先申请一个vector，用于存储各个线性链表的头指针各个线性链表又称为桶，每个桶都有自己的编号，通常是数组下标，当有新的键值对（元素）存储时，就按照哈希表的操作执行即可。

另外，因为底层采用的是哈希表，所以也有一个哈希表的负载因子的概念。这个概念用于衡量容器存储键值对的空/满状态，负载因子越大，意味着容器越满，即各链表中挂载着越多的键值对，这无疑会降低容器查找目标键值对的效率；反之，负载因子越小，容器越空，但并不一定各个链表中挂载的键值对就越少。

默认情况下，STL标准库对无序关联式容器的最大负载因子限制为1.0，当超过时，就会自动增加桶个数，同时对桶内键值对（元素）进行再哈希，此过程会导致部分容器迭代器失效，但指向单个键值对的引用或者指针仍然有效

2.和关联式容器相比，无序容器擅长通过指定键查找对应的值（平均时间复杂度为 O(1)）；但对于使用迭代器遍历容器中存储的元素，无序容器的执行效率则不如关联式容器

内容

常用的无序关联容器通常包括如下容器：

unordered_map 存储键值对 ，要求各个键值对键的值key不允许重复，且该容器中存储的键值对是无序的
unordered_set 存储元素，是无序且不重复的，相当于unordered_map中键值对的key=value
unordered_multimap 和 unordered_map 容器唯一的不同是 unordered_multimap 容器中存储元素的键可以重复（一旦值重复，则意味着键也是重复的）
unordered_multiset 和 unordered_set 容器唯一的不同是 unordered_multiset 容器中存储元素的值可以重复

无序关联式容器迭代器的通用函数

由于无序关联式容器都支持前向迭代器，无序关联式容器支持的操作符包括++p,p++,*p,==和!=,并且对于任意一个关联式容器都支持如下四个操作

begin() 返回指向第一个元素的前向迭代器
end() 返回指向容器最后一个元素之后一个位置的前向迭代器
cbegin() 和 begin() 功能相同，只不过在其基础上增加了 const 属性，不能用于修改元素
cend() 和 end() 功能相同，只不过在其基础上，增加了 const 属性，不能用于修改元素

需要注意的是，在操作无序关联式容器的过程（尤其是向容器中添加新键值对）中，一旦当前容器的负载因子超过最大负载因子，容器就会适当增加桶的数量，并自动执行 rehash() 成员方法，重新调整各个键值对的存储位置（此过程又称“重哈希”），此过程很可能导致之前创建的迭代器失效。需要指出的是，上面四个函数返回的迭代器不受影响

创建unordered_map及初始化：

成员函数

find(key) 在容器中查找键值为key的键值对，如果成功找到，则返回指向该键值对的前向迭代器；反之，则返回和 end() 方法一样的迭代器。另外，如果 unordered_map 容器用 const 限定，则该方法返回的是 const 类型的前向迭代器。
equal_range(key) 该方法返回一个 pair 对象（包含 2 个前向迭代器），表明当前容器中键为 key 的键值对所在的范围
empty() 若容器为空，则返回 true；否则返回 false
size() 返回当前 unordered_map 容器中存有键值对的个数
max_size() 返回 unordered_map 容器所能容纳键值对的最大个数，其返回值与操作系统相关
operator[] [] 运算符的重载，unordered_map[i]相当于获取对应unordered_map容器中key为i的键值对的value，另外当unordered_map容器中不包含键值为key的键值对的时候，使用[]运算符会直接往里面添加键值为key的键值对
at(key) 找到 unordered_map 容器中 key 键对应的值，如果找不到，该函数会引发 std::out_of_range 异常 这是at函数与[]最大的不同
insert() 向 unordered_map 容器中插入键值对
erase() 删除 unordered_map 容器指定位置、指定键（key）值或者指定区域内的键值对
swap() 在 2 个unordered_map容器的类型一致的条件下，交换 2 个 unordered_map 容器中存储的键值对
clear() 清空 unordered_map 容器中所有的键值对
emplace() 在当前 unordered_map 容器中的指定位置处构造新键值对。其效果和插入键值对一样，但效率更高
emplace_hint() 在本质上和 emplace() 在 unordered_map 容器中构造新键值对的方式是一样的，不同之处在于，使用者必须为该方法提供一个指示键值对生成位置的迭代器，并作为该方法的第一个参数
count(key) 在当前 unordered_map 容器中，查找键为 key 的键值对的个数并返回。此外，由于 unordered_map 容器中各键值对的键的值是唯一的，因此该函数的返回值最大为 1
bucket_count() 返回当前容器底层存储键值对时，使用桶（线性链表）的数量
max_bucket_count() 返回当前系统中，unordered_map 容器底层最多可以使用多少桶
bucket_size(i) 返回第 i 个桶中存储键值对的数量
bucket(key) 返回以 key 为键的键值对所在桶的编号
load_factor() 返回 unordered_map 容器中当前的负载因子
max_load_factor() 返回或者设置当前 unordered_map 容器的负载因子
rehash(n) 将当前容器底层使用桶的数量设置为 n
reserve() 将存储桶的数量（也就是 bucket_count() 方法的返回值）设置为至少容纳count个元（不超过最大负载因子）所需的数量，并重新整理容器
hash_function() 返回当前容器使用的哈希函数对象

unordered_set

直译为“无序的 set 容器”，是的，就是这样~

正如map和set的区别，unordered_map与unordered_set的区别是一样的

1.不再以键值对的形式存储数据，而是直接存储数据的值

2.容器内部存储的各个元素的值都互不相等，且不能被修改

3.不做排序处理

自定义哈希函数与比较函数

当我们的无序关联式容器需要存储一个自定义的结构体or类的时候，内置的哈希函数和比较函数就显得不够用了，此时我们就需要自定义哈希函数和比较函数。注意，我们自定义的哈希函数与比较函数都应该是函数对象。

函数对象

什么是函数对象？简单来说，一个类或者结构体重载了()，那么这个结构体或者类就称为一个仿函数，表示这个结构体或者类的实例化对象如果调用了()运算符，就会执行重载了的()的操作，但是它的调用形式又很像一个对象，而实际上它执行了一个函数的操作，所以被称为仿函数，又称函数对象
