1. intro

1.1 图神经网络

图神经网络，研究的是如何将神经网络应用到图上

通过图神经网络，我们将不再需要人为手动进行特征工程$(feature$ $engineering)$的相关操作，转而使用表示学习$(representation$ $learning)$的方式进行

通过图表示学习，将图的信息（可以是结点，子图，边等）表示为向量/嵌入$(embedding)$ ，图中相似的结点的嵌入结果也会相似

1.2 任务

目前，图神经网络的主要任务可以分为四大类：

  1. 节点级别 $node$ $level$
  2. 边级别 $edge$ $level$
  3. 社区 / 子图级别 $community(subgraph)$ $level$
  4. 图级别，包括预测任务 $graph-level$ $prediction$ 和 图生成任务 $graph$ $generation$

2. 传统的图机器学习

传统机器学习过程可以描述为：设计并获取所有训练数据上节点/边/图的特征→训练机器学习模型→应用模型

图数据本身就会有特征，但是我们还想获得说明其在网络中的位置、其局部网络结构$local\ network\ structure$之类的特征（这些额外的特征描述了网络的拓扑结构，能使预测更加准确）
所以最终一共有两种特征：数据的$structural\ feature$，即图，以及其本身的$attributes\ and\ properities$，即对应的结点/边/图的属性

本章重点着眼于传统的图机器学习方法

图机器学习的目标：对一系列 $object$ 做预测
$design\ choice:$
$Features:\ d-dimensional\ vectors$
$Objects:\ Nodes,\ edges,\ sets\ of\ nodes,\ entire\ graphs$

2.1 Traditional Feature-based Methods: Node

2.1.1 Node degree

以点的度作为描述特征，会导致相同度数的点被错误分类

2.1.2 Centrality

度中心性 $Node\ Centrality$

度中心性的计算方式为 $\frac{k_v}{N - 1}$， $k_v$ 为结点 $v$ 的度数

特征向量中心性 $Eigenvector\ Centrality$

认为如果节点邻居重要，那么节点本身也重要

这是个递归式，解法是将其转换为矩阵形式可得 $\lambda c=Ac$
从而发现 $c$ 就是特征向量。根据 $Perron-Frobenius\ Theorem$ 可知最大的特征值 $\lambda_{max}$ 总为正且唯一，对应的 $leading\ eigenvector\ c_{max}$ 就是 $c$ 向量

中介中心性 $Betweenness\ Centrality$

中介中心性考虑的是一个结点作为中介（类似图论中的桥）的可能性，认为如果一个节点处在很多节点对的最短路径上，那么这个节点是重要的。

注：实例中的 $c_v$ 算错了

连接中心性 $Closeness\ Centrality$

连接中心性描述的是一个结点作为图中心的程度，越小越接近图中心

$Clustering\ Coefficient$ 衡量节点邻居的连接程度
描述节点的局部结构信息，相同的节点对只算一次

由于其描述的是结点 $v$ 的邻居之间的关系，可以形象的看成是找三角形

2.1.3 Graphlet

$Graphlets$ 是一组有根连接的诱导非同构子图 $Rooted\ connected\ induced\ non-isomorphic\ subgraphs$

$induced\ graph$ 即从图中选一些点，然后把连接这些点的所有边也一起选上构成的图

$isomorphic\ graph$ 即有一些图看起来形状不一样，但是实质上其结点、连接方式相同，这样的图彼此是同构的

对于 2-5 个结点，可以有这样的一些 $Graphlets$

$GDV$ 即给出一组 $Graphlets$ ，指定图中某个节点，求这个结点对应的有多少个对应的 $Graphlets$ 的根的数目

注：$c$ 为 0 是因为以 $u$ 为根时得到的子图不是 $induced$ 的

2.1.4 Summary

总的来说，传统方法可以分为两类：

对结点的重要性进行分级

对结点的拓扑/结构进行分级

2.2 Traditional Feature-based Methods: Link

对于边的预测，是基于已经存在的边，预测新的边。模型运行时将会对点对进行排序，取最高的 $k$ 个点对作为结果。

对于边的预测可以分为两类

1.随机缺失边

2.随时间演化边

对于边的预测通常采用相似性算法，计算出点对之间的相似性分数 $C(x,y)$，按照该分数排序，选择得分最高的 $n$ 个点对作为输出

2.2.1 相似性分数

相似性分数的选择主要分为三类

基于距离 $Distance-based\ feature$

采用节点之间的最短距离计算点对分数，缺点是没有考虑到这条最短路径的性质（比如最短路长度相同，有多条最短路的点对肯定比最短路数目少的点对的稳定性更好）

捕获结点的共同邻居数 $Local\ neighborhood\ overlap$

这种点对的分数计算有三种方式：

1.计算点对 $v_1$ 和 $v_2$ 之间的交集个数

2.计算点对 $v_1$ 和 $v_2$ 之间的交集个数除以计算点对 $v_1$ 和 $v_2$ 之间的并集个数

3.$\sum{u\in N(v_1)\cap N(v_2)}\frac{1}{\log{k_u}}$

但是，这种计算方式也有缺点，比如一个点对之间没有共同邻居，但是这个点对之间的距离是3（即点对之间只差两个点），这样的点对在未来也很有可能会产生连接，为了克服这个问题，提出了全局共同邻居

全局共同邻居 $Global\ neighborhood\ overlap$

全局共同邻居，通过计算所有长度的点对之间的路径数目作为分数

$Katz$ 需要计算长度为 $l$ 的路径数目，这可以通过计算 $A^k$ 得到

根据矩阵理论，$Katz\ index$ 的计算方式为 $S_{v_1v_2} = \sum^{\infin}_{l=1}\beta^lA^l_{v_1v_2}$ 其解析解为 $(I-\beta A)^{-1}-I$ ，其中 $\beta$ 是一个系数，会给比较长的距离以比较小的权重

2.3 Traditional Feature-based Methods: Graph

图级别特征构建目标：找到能够描述全图结构的特征

2.3.1 Kernel

为了计算全图的结构特征，我们可以使用 $Kernel$

两个图的核 $K(G,G’)$ 以标量衡量其相似度，存在特征表示 $ϕ ( ⋅ )$ 使得 $K ( G , G’) = ϕ ( G )^T ϕ ( G’) $

定义好核后就可以直接应用 $SVM$ 之类的传统机器学习模型。
$ϕ ( ⋅ )$ 是个表示向量，不需要被显式地计算出来

下面主要介绍两种核

不管是什么核，核心思想都是用 $BoW$ 的形式，计算 $bag-of$

2.3.2 Graphlet Kernel

$Graphlet\ Kernel$ 计算图当中的不同 $Graphlet$ 的个数

注：此处的 $Graphlets$ 和上文的 $Graphlets$ 有不同，这里允许 $Graphlets$ 不需要是连通的，且是无根的

一组 $Graphlets$ 的实例

假定我们有一组 $Graphlets$ ，我们使用 $Geaphlets\ Kernel$ 计算向量 $f_G$

一个 $f_G$ 计算的实例

计算出 $f_G$ 和 $f_{G’}$ 后，进行点积运算即可直接得到 $K(G,G’)$，需要注意的是，如果 $G$ 和 $G’$ 之间的尺寸差距比较大，可以进行归一化

但是，这种计算方式会带来高昂的计算代价，为了解决这一问题，提出了 $Weisfeiler-Lehman\ Kernel$
