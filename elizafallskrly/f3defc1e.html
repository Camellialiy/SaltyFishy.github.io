0.机器学习 Machine learning

A computer program is said to learn from experience E with respect to some task T and some performance measure P, if its performance on T, as measured by P, improves with experience E

1.监督学习 Supervised learning

已知数据的label

回归问题：回归问题是预测连续值的输出，例如预测房价。

分类问题：分类问题是预测离散值输出，例如判断肿瘤是良性还是恶性。

2.无监督学习 Unsupervised learning

无监督学习即没有label，数据缺乏明确含义，比如给定一些数据但不知道它们具体的信息，对于分类问题无监督学习可以得到多个不同的聚类，从而实现预测的功能。

3.线性回归 Linear regression

通过线性函数将数据拟合为一条直线，多个变量对应多元线性回归。

3.1 代价函数 Cost function

代价函数，一般用于计算预测值 $h_\theta(x)$ 与 $y$ 之间的代价，通常使用最小均方差。

3.2 凸函数 Convex function

对于一般的最优化问题，如果为凸函数，则通过最小化代价函数求得最值。

3.3 梯度下降 Gradient descent

梯度下降给出了求解 $\theta_i$ 的方法。

梯度下降法需要同时更新所有参数

梯度下降的核心即代价函数对 $\theta_i$ 的偏导数和学习率 $\alpha$

偏导数，用来计算当前参数对应代价函数的斜率，导数为正则 $\theta_i$ 减小，导数为负则 $\theta_i$增 大，通过这样的方式可以使整体向 $\theta_i = 0$ 收敛。

$\alpha$ 用来描述学习率，即每次参数更新的步长。$\alpha$ 的大小不好确定，如果太小则需要很多步才能收敛，如果太大最后可能不会收敛甚至可能发散。

即使学习率不变，梯度下降也会自动收敛，因为偏导会一直减小。

梯度下降的本质，就是不断重复计算并同时更新偏导

虽然梯度下降可能会导致陷入局部最优解，但是如果求解的是凸函数，那么问题就迎刃而解了

有时候我们的特征是多元的

此时应当重新构建线性函数 $h_\theta(x)$，为了统一变量构建向量 $x$，引入 $x_0 = 1$

多元梯度下降的偏导计算只需要对其他的 $\theta_i$ 求导

3.4 特征缩放 Feature Scaling

不同特征的度量不同，可能导致学习效果不佳，通过特征缩放将其调整为一个大致相同的范围。

一般缩放到 $-3 \leq x \leq 3$ 或 $-\frac{1}{3} \le x \le \frac{1}{3}$

特征缩放有许多种方法：

标准化： $x_i = \frac{x_i - \mu}{\sigma}$ ，符合 $N(0,1)$

归一化：$x_i = \frac{x_i - min(x)}{max(x) - min(x)}$

平均归一化：$x_i = \frac{x_i - mean(x)}{max(x) - min(x)}$

3.5 学习率 Learning rate

学习率 $\alpha$ 的大小会影响梯度算法的执行，太大可能会导致算法不收敛，太小会增加迭代的次数。

学习率太小，将导致梯度下降缓慢，学习率太大将导致无法收敛。

9.2 偏差、方差 Bias/Variance

随着多项式次数的增加，训练集的误差慢慢变小，但是验证集的误差变大，分别对应着欠拟合和过拟合

对于高方差，即过拟合，此时的 $h_{cv}(\theta)$ 高但 $h_{train}(\theta)$ 低

9.4 学习曲线 Learning curves

随着数据量的增加，$J_{train}(\theta)$ 的误差慢慢增大，因为数据越少，模型越容易拟合；$J_{cv}(\theta)$ 慢慢减少，因为数据越多，模型越精准，所以误差减小。

高偏差

因为参数很少，数据很多，所以随着数据的增多高偏差的模型的 $J_{train}(\theta)$ 和 $J_{cv}(\theta)$ 很接近。这时选择增加数据就不是很好的选择了，可以尝试增加数据的特征

高方差

高方差的特点是训练误差和验证误差之间有很大的差距，这时可以选择增加数据，随着图像右移训练误差和验证误差会慢慢接近

10.1 代价函数 Cost function

支持向量机的代价函数与逻辑回归十分相似，对于 $y=1$ 的情况，称为 $cost_1(z)$ ，而 $y=0$ ，称为 $cost_0(z)$ ，其图像如下图所示

对于支持向量机的代价函数，删除 $\frac{1}{m}$ 和 $\lambda$，并替换 $logh_\theta(z)$ 为 $cost_{y^{i}}(z)$，形式写为 $CA+B$

10.2 大间距 Large margin

要令其代价函数最小，则需要在 $y^i=1$ 时令 $\theta^Tx^i\ge1$ 当 $y^i=0$ 时令 $\theta^Tx^u\le-1$

支持向量机的决策边界与样本点的距离称为 $margin$ ，与所有样本点距离最大的就是决策边界

对于支持向量机，其参数 $C$ 决定了决策边界的划分，若 $C$ 很大，则有可能呈现斜线（过拟合），可以适当减小 $C$ 使得变为直线

10.3 数学机理 mathematics

向量的内积等于其投影长度的乘积，因此对于 $\theta^Tx$ 可以写为 $x||\theta||$

对于左侧的 $\theta$，每个 $x$ 向量在 $\theta$ 上的投影距离很小，要满足条件那么 $||\theta||$ 就要增大，这样最小化代价函数的值就不是最优的，可能发生过拟合，所以 $SVM$ 会得到类似右侧的边界，这样 $||\theta||$ 尽可能小些，因为投影距离都比左侧的要大，这也是最大边界的原因。

10.4 核函数 Kernel

当我们考虑一个非线性决策边界问题时，我们可能会使用高阶函数进行拟合，但是可能存在比我们的假设更好的表示形式

我们可以将每种特征表示为$f_i$，使用高斯核函数来做相似度分析

使用高斯函数的特点：如果相似度很高，即对应的欧几里得距离$\approx 0$对应$f_i=1$，相反如果相似度低对应$f_i\approx 0$

对于已经确定的 $\theta$ ，使用新构建的特征 $f^i$ 进行预测

对于核函数， $l^i$ 即为 $x^i$ ，于是可以计算 $f^i$

所以目标函数也应当一并更新

类似目标函数，核函数的 $\sigma^2$ 决定了拟合程度

14.1 高斯分布 Gaussian distribution

通常，我们可以使用高斯分布来进行以异常检测。

对于每一个特征都分别计算 $\mu$ 和 $\sigma^{2}$ ，得到 $n$ 个特征的高斯分布

从而对于每一个样本点，把 $n$ 个概率相乘，就可以得到对应的概率 $p(x^{i})$，即对任意 $i \in [1,m]$ $p(x^{i}) = \prod_{j=1}^{n}p(x^{i}_{j};\mu_{j},\sigma^{2}_{j})$

然后我们就可以根据设定的阈值 $\epsilon$ 确定是否出现了异常

从二维图上看，就像是划定了一个范围，范围之外的都是异常区间。

16. 大数据集 Big dataset

16.1 随机梯度下降 Stochastic gradient descent

对于普通的梯度下降算法（以线性回归为例），每次迭代我们会计算所有的 $cost$ ，这需要 $m$ 次求和，然后批量更新参数 $\theta$ ，这可能导致庞大的计算量。

我们可以修改我们的做法，采用随机梯度下降，这种做法首先随机打乱数据。然后我们遍历特征，对于每一个特征，都执行一次参数 $\theta$ 的更新，这样每次 $m$ 个特征执行完，参数 $\theta$ 也被更新了 $m$ 次。

通过 $SGD$ 得到的结果不一定比 $GD$ 好，但是好处是运行的更快。

如何判断 $SGD$ 是否已经收敛？

在已知的 $cost$ 函数的基础上，不断迭代，每迭代一定次数进行绘图，根据均值判断。

如果图像上下震荡，说明噪声太多，那么我们应该减小迭代次数，

如果随着迭代次数 $cost$ 上升，我们应该减小 $\alpha$

$SGD$ 一般不能得到全局最优，他会一直在最优值附近徘徊。

学习率的大小一般保持不变，一个思路是可以动态的改变学习率 $\alpha$ 的大小来提高准确度，比如随着迭代次数的增加慢慢减小 $\alpha$ 的值。

16.2 Mini-batch

对于 $SGD$ 如果我们修改每次迭代过程中同时更新的参数 $\theta$ 的数量，即一次同时计算多个特征进行参数 $\theta$ 的梯度下降，就称为 $Mini-batch$，其中 $b$ 表示选择的特征个数

一般可以选择的 $b$ 的范围在 $[2,100]$
